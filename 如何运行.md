# Nexus Lab Demo 运行指南

本文档说明如何在本地运行 Nexus Lab Demo 项目。

---

## 一、环境准备

### 1. 安装 Node.js

本项目需要 **Node.js**（自带 npm），不能通过 Python 的 `pip install npm` 安装。

- **Windows**  
  - 打开 https://nodejs.org/ ，下载 **LTS** 版本并安装。  
  - 或使用：`winget install OpenJS.NodeJS.LTS`  
  - 安装后**重新打开终端**。

- **验证**：在终端执行以下命令，能显示版本号即表示安装成功。
  ```bash
  node -v    # 例如 v20.x 或 v22.x
  npm -v     # 例如 10.x
  ```

### 2. 获取 Second Me 应用密钥

1. 打开 [Second Me 开发者平台](https://develop.second.me/)。
2. 登录/注册后，创建应用。
3. 在应用详情页获取：
   - **Client ID**
   - **Client Secret**
4. 在应用设置中配置「重定向 URI」：
   - 本地：`http://localhost:3000/api/auth/callback`
   - 若已部署线上：`https://你的域名/api/auth/callback`

## 二、本地运行步骤

### 步骤 1：进入项目目录

在终端中执行（路径请按你本机实际存放位置调整）：

```bash
cd C:\Users\Administrator\Desktop\A2A\nexuslab-demo
```

### 步骤 2：配置环境变量

1. 复制示例文件：
   ```bash
   copy .env.local.example .env.local
   ```
   （Linux/macOS 用：`cp .env.local.example .env.local`）

2. 用任意文本编辑器打开 `.env.local`，填入你的 Second Me 密钥：

   ```env
   SECONDME_CLIENT_ID=你的Client_ID
   SECONDME_CLIENT_SECRET=你的Client_Secret
   SECONDME_REDIRECT_URI=http://localhost:3000/api/auth/callback
   ```

   注意：不要将 `.env.local` 提交到 Git，该文件已在 `.gitignore` 中。

### 步骤 3：安装依赖

在项目目录下执行：

```bash
npm install
```

等待依赖安装完成（首次可能稍慢）。

### 步骤 4：启动开发服务器

```bash
npm run dev
```

看到类似以下输出即表示启动成功：

```
▲ Next.js 14.x.x
- Local:        http://localhost:3000
```

### 步骤 5：在浏览器中访问

1. 打开浏览器，访问：**http://localhost:3000**
2. 点击「使用 Second Me 登录进入实验室」。
3. 使用 Second Me 账号完成授权后，会进入「实验室」页面，可体验碰撞话题、匹配 Agent、会诊入口等。

## 三、常用命令

| 命令 | 说明 |
|------|------|
| `npm install` | 安装项目依赖（首次或依赖变更后执行） |
| `npm run dev` | 启动开发服务器（默认 http://localhost:3000） |
| `npm run build` | 构建生产版本（部署前可本地验证） |
| `npm run start` | 使用构建结果启动（需先执行 `npm run build`） |

## 四、部署到线上（如 Vercel）

1. 将本仓库或 `nexuslab-demo` 目录连接到 [Vercel](https://vercel.com/)。
2. 在 Vercel 项目 **Settings → Environment Variables** 中配置：
   - `SECONDME_CLIENT_ID`
   - `SECONDME_CLIENT_SECRET`
   - `SECONDME_REDIRECT_URI` = `https://你的Vercel域名/api/auth/callback`
3. 在 Second Me 开发者平台中，为该应用添加重定向 URI：`https://你的Vercel域名/api/auth/callback`。
4. 部署完成后，用生成的 URL 作为比赛提交的「可访问 Demo 地址」。

## 五、常见问题

**Q：终端里执行 `npm` 提示“无法识别”或“不是内部或外部命令”？**  
A：说明当前终端没有识别到 Node.js/npm。请确认已安装 Node.js，并**重新打开终端**（或重启 Cursor/IDE）后再试。

**Q：登录时提示 redirect_uri 错误？**  
A：检查 Second Me 应用里的「重定向 URI」是否与 `.env.local` 中的 `SECONDME_REDIRECT_URI` 完全一致（含 `http://` 或 `https://`、端口、路径）。

**Q：想用知乎热榜等能力申报知乎特别奖？**  
A：在赛事群申请知乎 AK/SK 后，在 `.env.local` 中配置 `ZHIHU_ACCESS_KEY`、`ZHIHU_SECRET_KEY`，并在代码中按知乎接口文档接入（如 `src/app/api/zhihu/billboard/route.ts`）。
